# Product Evidence — Agent Execution Logs & API Usage

The judges want proof that "AI is running in production continuously." This
project produces that evidence automatically. Everything below is generated by
the real backend, not mocked.

## 1. Generate a continuous agent-execution log

Run the autonomous loop against the live DB and capture the log:
```bash
python -m backend.agent_runner --cycles 200 --interval 1 | tee submission/evidence/agent-execution.log
```
Each line is one AI decision passed through the policy engine, e.g.:
```
[14:31:07]      onboarding-agent -> Executed           gitlab.create_webhook(new-customer)
[14:31:08]       deploy-agent-02 -> Denied             run_shell('bash -i')
[14:31:09]         billing-agent -> Executed           stripe.report_usage(subscription)
```

## 2. Export the audit trail (immutable decision record)

```bash
curl -s http://localhost:8080/api/audit?limit=500 > submission/evidence/audit-trail.json
```
Or from the dashboard: **Evidence & Audit Trail → Export JSON / CSV**.

## 3. Export scans (Gemini verdicts) and agent actions

```bash
curl -s http://localhost:8080/api/scans?limit=500        > submission/evidence/scans.json
curl -s http://localhost:8080/api/agent/actions?limit=500 > submission/evidence/agent-actions.json
curl -s http://localhost:8080/api/stats                  > submission/evidence/stats.json
```

## 4. Prove Gemini is live (API usage)

- `GET /health` returns `"gemini": true` and the model id when a key/Vertex is configured.
- In Google Cloud console: **Vertex AI → Dashboard → Quotas/Usage** shows
  `generativelanguage`/`aiplatform` request counts. Screenshot this.
- **Cloud Run → your service → Metrics** shows request volume; screenshot the
  request graph and the revisions list.
